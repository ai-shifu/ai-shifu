# 课后反馈弹窗时机

## 核心规则

- 自动弹窗必须满足“当前会话已滚动到底部”。
- 播放中进入 pending，待音频空闲后延迟约 1.2 秒再展示。
- 用户主动关闭后按 `outlineBid` 记录免打扰，不在同课节重复自动弹窗。

## 工作流

1. SSE 到达后先校验滚动门禁。
2. 若音频正在播放或播放序列未结束，转入 pending。
3. 播放空闲后启动延迟定时器再展示。
4. 关闭弹窗时写入 `outlineBid` 级别免打扰缓存。
5. 检测到已有有效评分（1-5）时标记已反馈，不再自动弹出。
